In this post, I walk through building a full-stack project combining data engineering and machine learning to forecast house prices in Australia.

Project Overview

Goal: Extract, transform, and load Australian house price time series into PostgreSQL, then train forecasting models and build a dashboard to visualize predictions.

Tech Stack: Python, Pandas, PostgreSQL, Prophet, XGBoost, Streamlit

Data Engineering: ETL Pipeline

I used the ABS Residential Property Price Indexes dataset, with historical median prices by capital city (quarterly data). It was downloaded and processed into a clean format.

extract.py loads the raw XLSX; transform.py cleans and feature-engineers the time series; load.py uploads everything into PostgreSQL.

Machine Learning

With clean data in PostgreSQL, I retrived it in Python for modeling:

  • Prophet: Time-series forecasting per state or capital city.
  • XGBoost: Regression using lag features, macro time-index variables.

Forecast Storage & Dashboard

Predictions from both models are stored in a price_forecasts table in PostgreSQL. Using Streamlit, I built an interactive dashboard to select state or suburb, view historical vs forecasted trends, and compare models.

Results & Key Learnings

  • Built automated ETL pipeline with proper validation
  • Achieved accurate forecasts at both state and suburb levels
  • Learned end-to-end integration of engineering and ML workflow
